## 一、观众角色与权限

一个 Table（房间）可以被四种用户访问。

### 1.1 角色定义

| 角色 | 是谁 | 如何成为这个角色 |
|---|---|---|
| **Host（房主）** | 创建这个 Table 的人 | 通过 "+ Open table" 入口开房 |
| **Member（朋友）** | 房主显式邀请加入的用户 | 通过房主分享的邀请链接加入 |
| **Spectator（围观群众）** | 任何已登录但不是 Member 的用户 | 已登录状态下打开 Table 链接 |
| **Visitor（访客）** | 任何未登录用户 | 没有 LokaCup 账号 / 已退出登录 |

### 1.2 权限矩阵

| 能力 | Host | Member | Spectator | Visitor |
|---|:---:|:---:|:---:|:---:|
| 查看 AI 辩论流 | ✅ | ✅ | ✅ | ✅ |
| 查看胜率曲线 / AI 共识结论 | ✅ | ✅ | ✅ | ✅ |
| 查看 BasicDataTabs（AI 分析 · 比赛数据 · 市场） | ✅ | ✅ | ✅ | ✅ |
| 查看右侧观众聊天 | ✅ | ✅ | ✅ | ✅ |
| 向 AI 团队提问（输入框 + 语音 mic） | ✅ | ✅ | ❌ | ❌ |
| Manage 弹窗（开关 AI analyst · 邀请/移除朋友） | ✅ | ❌ | ❌ | ❌ |
| Invite 邀请朋友 | ✅ | ❌ | ❌ | ❌ |
| Share 分享按钮 | ✅ | ✅ | ✅ | ✅ |
| 锁定预测 | ✅ | ✅ | ✅ | ❌ → 触发登录弹窗 |
| 在观众聊天里发言 | ✅ | ✅ | ✅ | ❌ → 触发登录弹窗 |
| Hover 消息复制按钮 | ✅ | ✅ | ✅ | ✅ |

### 1.3 实现规则

- Visitor 的写操作（投票、聊天）不静默失败 —— 弹出 LoginModal
- Host 独占的功能（Manage、Invite）对非 Host **直接隐藏**，不灰显
- Spectator 模式下，底部输入框替换为提示条：`Only members can prompt the AI`，语音 mic 隐藏
- Spectator 房间顶部显示 `👁 Spectating` 状态 pill

## 二、投票模型

### 2.1 基本规则

- 每个 Table 只有 1 个投票，由 Host 在开 Table 时设置
- 投票来源不限：Polymarket / Kalshi 链接 **或** Custom 问题 + 自定义选项，两种都赚积分
- 结算时间：比赛结束 / 外部市场结算

### 2.2 防刷限制

- 每个用户每天只能开 1 个 Table
- Host 跟普通用户一样参与自己桌的投票，无额外分成
- 不做连胜加成、AI 即时投票、比赛中加题

## 三、比赛生命周期状态

每个 Table 根据所属比赛分为三种状态：Upcoming（赛前）/ Live（直播中）/ Finished（已结束）。

### 3.1 AI 触发机制（所有状态共用）

AI 只有两种触发器：
1. **第三方数据源喂入新消息** —— 后端按比赛状态调整喂数据频率
2. **用户提问** —— 用户在输入框发问

不存在自我循环或定时 idle 输出。

后端喂数据的频率：
- 赛前：12–24 小时间隔
- 赛中：缩短间隔，更频繁
- 已结束：停止喂入

### 3.2 Upcoming（赛前）

- AI 行为：Table 开启时跑一轮 Initial Round，后续只响应触发器
- 投票：可投，到开赛时间自动锁单

### 3.3 Live（直播中）

- AI 行为：持续响应触发器输出
- 投票：可投，按当前 multiplier 结算

### 3.4 Finished（已结束 / 已结算）

- AI 行为：永久停止
  - 第三方数据源不再喂入
  - 用户提问不触发 AI 回复
  - 用户仍可在评论流发言，只是用户之间对话
- 投票：自动结算
  - 预测正确：发放积分
  - 预测错误：0 积分
  - 未投票：不计
