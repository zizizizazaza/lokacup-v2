# LokaCup 积分系统设计文档

> 版本：v1.1 · 草案 · 2026-06-04
>
> 范围：积分获取、计算公式、排行榜、抽奖、反作弊。
>
> 设计目标：**鼓励"预测 / 主持 / 社交"三条并行路径**，没有单一行为占主导，用户走任意一条都有竞争力。

---

## ⚠️ 文档读者须知（重要）

> **这份文档是产品 / 开发团队内部的计算规则**，**不直接展示给最终用户**。
>
> 用户能看到的只是：
>
> - 一个简化的"如何赚积分"页面，**仅列出 5 类行为 + 一句描述**（例如"投对预测拿分"、"开 Table 邀请人围观拿分"），**不暴露公式、系数、封顶**
> - 投票时的实时提示："投这边 +约 X pt"（**只显示估算值**，不解释为什么）
> - 结算时的 toast：`+45pt · Sharp Insight`（只给结果，不解释拆分）
> - Profile 页可见自己各大类的累计分，**但看不到难度系数 / 时机系数 / streak 倍率的具体数字**
>
> 复杂度全部留在后端。用户体验是"做事 → 拿分 → 上榜"。
>
> 见 §6 [用户视角的可见性](#6-用户视角的可见性) 完整对照表。

## 1. 总览

用户通过五类行为累积积分，排行榜（Leaderboard）每周/每季结算，**Top 50 抽门票**。

### 五类积分来源

> 设计目标：**三条主路径并存**（预测 / 主持 / 社交），用户走任意一条都有竞争力上榜。投票路径权重最高但不压倒性。

| 来源 | 一句话 | 占典型重度玩家积分比例 |
|---|---|---|
| 🎯 **预测投票** | 在 Table 内对 AI 预测的问题投票，投对得分，按赔率 / 时机加权 | **~45%** |
| 🎤 **开 Table** | 主持 Table，按在线时长 / AI 互动 / 围观峰值复合计分 | **~25%** |
| 👥 **邀请好友** | 把好友拉进 Table 在线 ≥3 分钟 | ~10% |
| 🪪 **邀请码** | 拉新用户注册 LokaCup（产品级拉新） | ~10% |
| 📢 **Twitter 分享** | 把 Table / 预测卡片分享到 Twitter | ~10% |

## 2. 详细规则

### 2.1 预测投票（核心机制）

**公式**：

```
得分 = 25 × 概率难度系数 × 时机系数
```

> 连胜 / Streak **不再参与积分计算**，但作为用户的**统计标签**保留（Profile 显示、可分享到 Twitter "5 连胜 🔥"）。理由：让积分公式更简洁可预期；连胜本身的故事性 / 炫耀价值靠社交分享体现，不靠积分膨胀。

**概率难度系数**（按投票当时 AI 给你那一方的胜率分档）：

| AI 胜率 | 系数 | 含义 |
|---|---|---|
| > 65% | 1.0× | "AI 看好的方向你跟上了" |
| 40 – 65% | 2.0× | "胶着局判断对" |
| 20 – 40% | 2.5× | "看到 AI 没看到的，但不算极端" |
| < 20% | 3.0× | "黑马预测家"（系数上限） |

**时机系数**：

| 时机 | 系数 | 设计意图 |
|---|---|---|
| 比赛开始前 | 1.5× | 鼓励真预测 |
| 0' – 60' | 1.0× | 标准 |
| 60' – 80' | 0.6× | 看牌猜结果，递减 |
| 80'+ 或一球内分差 | 0.3× | 几乎已知答案，最低保底 |

**单次投票分值区间**：

| 场景 | 公式 | 得分 |
|---|---|---|
| 80 分钟才下注的明显热门 | 25 × 1.0 × 0.3 | **~8pt**（最低保底） |
| 中盘下注，胶着局，判断对 | 25 × 2.0 × 1.0 | **50pt**（常见） |
| 黑马 + 赛前下注（理论上限） | 25 × 3.0 × 1.5 | **~115pt** |

**结算时间**：比赛结束 / 问题被 AI 标记 resolved 时计算积分，发到用户钱包。

**结算通知**：弹出 toast，"+45pt · Sharp Insight" 或 "+15pt · Consensus pick"。

### 2.2 开 Table（4 维复合，可跨天累计）

> Table 可重复进入。主持人离开后可再回来继续。
>
> 用户只看到合计积分（"今天你的 Table +47pt"），**不暴露细分公式**。

| 维度 | 触发 | 单位分 | 日封顶 | 备注 |
|---|---|---|---|---|
| **A. 活跃在线时长** | 主持人在 Table 内**在线且有真实操作**（5 分钟无操作不计；活跃定义见下） | 每 10 分钟 **+8pt** | 120pt/天 | 防挂机，也防"刷消息薅羊毛" |
| **B. 围观峰值** | Table **历史最高同时在线**人数达到阈值（一次性领取，不可重复） | 见下表 | 一次性 | 长尾激励 |
| **C. 通过 Table 邀请到好友** | — | **合并到 §2.3** | — | 避免重复计分 |

**"活跃"判定**（任意一项即视为这 10 分钟在线活跃）：

- 鼠标 / 键盘有实际操作
- 主持人有发消息给 AI Agents
- 用户在 Table 内投票
- Table 触发了新一轮辩论（来自 AI / 比赛事件）

> ⚠️ 不再按"消息条数"计分（v1.1 设计），避免用户疯狂发垃圾消息薅羊毛。现在的逻辑是 **"你只要在场、有动作，就给固定时长积分"**，跟具体动作多少无关。

**围观峰值阶梯**：

| 同时在线 | 一次性奖励 |
|---|---|
| 10 人 | +25pt |
| 50 人 | +120pt |
| 100 人 | +300pt |
| 500 人 | +800pt |
| 1000 人 | +1,500pt |

> "同时在线" = 同一时刻独立账号数，**已 unique 去重**。

### 2.3 邀请好友进 Table

- 好友通过你的链接进入某个 Table，**在线 ≥3 分钟**：**+15pt / 人**
- **每周封顶 200pt**
- 同一好友进入同一 Table 多次只算一次
- 同一好友进入不同 Table 可分别计分

### 2.4 邀请码（拉新到 LokaCup）

- 新用户用你的邀请码完成注册：**+20pt**
- 该新用户**注册 7 天内完成一次投票**：**额外 +80pt**
- **无周封顶**（鼓励拉真活跃用户）

### 2.5 Twitter 分享

- 每次分享 Table 或自己的预测卡片：**+15pt**
- **每天只奖励第一次**，当天后续分享不再计分
- 日封顶 **15pt**

**分享卡片内容（必备）**：

- 主问题（"Brazil to win the World Cup?"）
- 用户的预测（"Predicted Brazil ✓"）
- 投票时的 AI 胜率（用于叙事，不暴露公式）
- 链接（带 UTM 跟踪用户邀请码）

**分享卡片叙事框架**（品牌一致性）：

- ✅ **跟 AI 同向赢**：
  > 🏆 *"Predicted: Brazil ✓ — Locked in with LokaCup consensus. +15 pts"*

- ✅ **反 AI 赢（黑马）**：
  > 🎯 *"Sharp insight: Field ✓"*
  > *"Called the upset 20 minutes before the model caught up."*
  > *"+45 pts · Sharp Insight bonus"*

- ❌ **绝对不要的措辞**：
  - "I beat the AI"
  - "AI said 30%, I knew better"
  - "Outsmarted LokaCup AI"

把"用户反 AI 赢"重新包装成 **"用户的判断让 AI 更聪明"** —— contrarian 投票真实地接入下一轮 Debate Rounds 的 input signal（"Sharp insight from user XYZ shifted Tactics Analyst confidence by 4pt"），叙事变成共同进步。

## 4. 排行榜与抽奖

### 4.1 排行榜结构

- **页面**：`/leaderboard`
- **类型**：**累计积分排行榜**（不按周 / 季周期重置）
- **结算时点**：积分实时累计，**只在赛事关键节点做快照 + 发奖**（见 §4.2）
- **展示**：Top 50 + 你当前名次 + 距离 Top 50 还差多少分

### 4.2 抽奖与奖品

- **结算节点**（基于赛事进程，**不按时间周期**）：
  - **1/4 决赛开赛前**：快照积分榜 → 抽 **1/4 决赛门票**（2 张）
  - **半决赛开赛前几天**：快照积分榜 → 抽 **半决赛门票（2 张）+ 决赛门票（1 张）**
- **资格**：每次快照时积分榜对应名次区间内的用户（具体见下表）
- **审核**：规则自动审核（见 §5）
- **方式**：**分层等概率抽奖**（Tiered Equal-Probability Lottery）

**奖品池结构**：

| 奖品 | 抽奖池 | 数量 | 单池中奖率 |
|---|---|---|---|
| 🏆 **决赛门票** | **Top 10** | 1 张 | 10% |
| 🥈 **半决赛门票** | **Top 30**（决赛中奖者除外）| 2 张 | ~7% |
| 🥉 **1/4 决赛门票** | **Top 50**（前两轮中奖者除外）| 2 张 | ~4% |

**抽奖规则**：

- 抽奖顺序：决赛 → 半决赛 → 1/4 决赛
- **已中奖者不进入后续抽奖**，避免一人通吃
- 每个池子内**等概率随机**，不按具体名次加权

**Top 50 内的体感分档**：

| 名次区间 | 拿奖概率 | 体感 |
|---|---|---|
| Top 1 – 10 | **21%**（决赛 10% + 半决 7% + 1/4 4%）| 每个池子都在 |
| Top 11 – 30 | **11%**（半决 7% + 1/4 4%）| 决赛没戏，但仍有大奖机会 |
| Top 31 – 50 | **4%**（1/4 4%）| 保底门票池，冲进 Top 30 解锁更多 |
| 50 名开外 | 0%（**但积分仍计入空投，见 §4.3**） | — |

> 这种三段式悬崖在赛季尾盘会触发集中冲分行为（30→29、50→49），是产品端预期的高流量峰值。

### 4.3 积分与 Setu 空投预期

**所有本季积分**（包括 50 名开外的）**会作为 Setu 项目后续空投的快照依据**。

意义：

- **50 名开外的用户也有动力持续累积积分** —— 即便本季抽不到门票，积分会沉淀为空投预期
- 整个积分体系从"赛季抽奖"扩展为 **"链上资产预期"**，长期价值更高
- 主推叙事：**"每一分都会算在你头上"**

> Setu 空投的具体规则、快照时点、token 比例由 Setu 项目方另行公布，本积分系统只负责积累与排名。

## 5. 防作弊（规则自动化）

| 检测项 | 处理 |
|---|---|
| 同 IP / 同设备指纹聚类账号 | 自动标记，超过阈值的账号本期积分清零 |
| 新账号注册 7 天内 | 积分**不进排行榜**（防小号挂机刷分） |
| 邀请关系成环（A→B→C→A）| 该环涉及账号本期积分清零 |
| 异常投票模式（短时间内高频投黑马且全对）| 自动降权 / 清零 |
| Twitter 分享链接无效（不可访问 / 已删除）| 该次分享不计分 |

> 所有审核走规则自动化，不依赖人工。

## 6. 用户视角的可见性

| 内容 | 用户看得到吗 |
|---|---|
| 总积分 / 当前名次 | ✅ Profile 页 |
| 距离 Top 50 还差多少分 | ✅ Leaderboard 顶部条 |
| 单次投票得了多少分 | ✅ 结算 toast + Profile 历史 |
| 每个行为大类的合计积分（投票 / 开 Table / 邀请 / 分享）| ✅ Profile 行为分解 |
| 当前 streak | ✅ Profile + 投票 UI 提示 |
| **难度系数 / 时机系数的具体数字** | ❌ 不展示，保持神秘感 |
| **开 Table 的 4 维拆分** | ❌ 只给合计分 |
| **反作弊规则** | ❌ 反作弊靠不透明 |
