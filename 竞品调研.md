## 一、市场分类

围绕"AI + 预测"目前市场可清晰分成两类，**它们的用户、数据栈、产品形态完全不同**：

| 类别 | 服务对象 | 数据基底 | 输出 |
|------|---------|---------|------|
| **A. AI 足球预测** | 传统 sportsbook 投注者 | 足球比赛数据（xG/阵容/伤病/赔率） | 1X2、比分、BTTS、大小球的概率 |
| **B. Polymarket 生态 AI** | 预测市场交易者 | CLOB 订单簿 + 新闻 + 链上数据 | 市场隐含概率 vs AI 估计概率的 edge |

**没有任何一个产品同时服务两边**——这是我们的产品空间。

## 二、核心竞品（足球 AI 预测类）

只挑 3 个最值得对标的：

### 2.1 [OddsFlow.ai](https://www.oddsflow.ai/) — **最直接的世界杯竞品**

- **做什么**：AI 价值投注工具，计算"公平赔率 vs 庄家赔率"，给出 edge %。已经为世界杯每场比赛建立独立预测页（如 `/zh/worldcup/predictions/2026-06-13/brazil-vs-morocco-1489371`），多语言（含中文）
- **数据栈**：FIFA 排名、ELO 评分、预选赛战绩、历史交手、阵容、**Bet365 / Pinnacle 实时赔率**
- **方法论**：单一 ML 模型；公开历史 performance logs，自报 ~54-56% 胜率、+17% ROI
- **定价**：订阅制
- **强**：透明（公开日志/PDF 记录）、实时 mispricing、专攻世界杯
- **弱**：界面偏技术、单模型、不接 Polymarket、不解释为什么

### 2.2 [NerdyTips](https://nerdytips.com/) — **数据栈最完整的足球 AI**

- **做什么**：自研 AI 引擎 NT Apex，160+ 联赛日推预测，Banker 高置信 tips
- **数据栈**：xG、控球、**球员转会、新闻**、状态、H2H；存储在自己的大数据平台
- **方法论**：Java + AI + 数学建模 + ML；三阶段：采集 → 归一化 → 预测；声称 25 万+ 场比赛历史
- **定价**：$5.99-$9.99/月
- **强**：覆盖广、Trustpilot 4.5/5、**是少数把新闻 + 转会纳入特征集的足球 AI**
- **弱**：单模型黑箱、无 Polymarket 整合、解释性一般

### 2.3 [Opta Supercomputer (theanalyst.com)](https://theanalyst.com/articles/opta-football-predictions) — **叙事霸主**

- **做什么**：媒体级世界杯预测文章，全球媒体引用基准
- **数据栈**：Opta Power Rankings（Elo 衍生：净胜球、赛事系数、攻防分项，250+ 联赛 13000+ 球队）+ **博彩市场赔率**
- **方法论**：单场胜/平/负概率 → 蒙特卡洛模拟剩余赛程 → 夺冠/出线 %
- **强**：**唯一方法论完全公开**的对手；可信度天花板；坦承赔率是模型输入
- **弱**：单向编辑稿，不可问、不可追问、不可个性化

> 其他足球 AI（Forebet、Footystats、KickOff.ai、Scoore.ai、SportBot AI、Predicd 等）方法论同质化严重，不一一列举。

## 三、核心竞品（Polymarket 生态 AI 类）

只挑 4 个最有代表性的：

### 3.1 [Polymarket Agents 官方](https://github.com/Polymarket/agents) — **参考实现**

- **数据**：Gamma API（市场元数据）+ CLOB（订单簿/成交）+ **NewsAPI → Chroma 向量库**做 RAG
- **方法论**：单 LLM + RAG；市场价 vs LLM 概率 → edge 下单
- **特点**：通用，不分场景；体育不是它的强项
- **借鉴价值**：**新闻 → RAG → LLM 估计概率**的标准链路

### 3.2 [Alphascope](https://alphascope.app/) — **研究情报型**

- **做什么**：粘贴 Polymarket 链接 → AI 自动汇总新闻、概率分析、催化剂扫描、钱包追踪、跨平台（Kalshi 等）对比
- **数据**：新闻聚合 + 链上巨鲸数据 + 跨市场赔率
- **方法论**：单 LLM 情报聚合，不强调辩论或共识
- **定价**：免费 beta + 付费
- **强**：上下文深度、新闻 + 催化剂、研究型交易者最爱
- **弱**：**无结构化多视角**，单 AI 摘要风险高

### 3.3 Polymarket Prediction Alert AI（App） — **picks 推送型**

- **做什么**：AI 选市场、日推 picks、智能警报、AI 概率 vs 市场价差、关键驱动因素
- **数据**：未完全公开，大概率是 LLM + 新闻 + 价格历史
- **方法论**：单模型 → actionable picks + 理由
- **强**：移动端、直接给结论 + 推送
- **弱**：理由可能较薄、单模型

### 3.4 [Willmin Kalshi-Claude Bot](https://github.com/Willmin-wm/kalshi-claude-trading-bot) — **唯一开源的多 Agent 共识实现**

- **数据**：仅 Kalshi 订单簿；**无外部数据**——三 Agent 完全靠 Claude 参数化知识
- **方法论**：Forecaster / Critic / Trader 三角色辩论 → 共识后交易 + Kelly sizing
- **强**：**架构上与我们最像**，必须读源码理解共识机制
- **弱**：数据极简、纯 Kalshi、未产品化

> 其他（PolyPulse / POLYBRO 浏览器扩展、Unusual Predictions 巨鲸追踪、Octagon Kalshi CLI、学术的 PolyBench / PolyGnosis 2.0）作为参考存在，但不是直接竞品。

## 四、我们的数据需求清单

> **前提**：我们做的是"世界杯比赛 → Polymarket 市场 → AI 多 Agent 共识预测 + 解释"。
>
> 把竞品的数据栈拼起来，可以得出我们**真正需要**接入的数据。

### 4.1 必备：Polymarket 市场数据（核心定价锚）

| 数据 | 用途 | 来源 |
|------|------|------|
| **市场元数据** | 知道有哪些世界杯市场、解算标准是什么 | [Polymarket Gamma API](https://docs.polymarket.com/developers/gamma-markets-api/overview) |
| **CLOB 订单簿** | 实时买卖一价 / 深度 / 点差 | [Polymarket CLOB](https://docs.polymarket.com/developers/CLOB/introduction) |
| **历史价格 / 成交磁带** | 看价格如何随消息/比赛进程变化、回测 | Gamma + CLOB 历史端点 |
| **巨鲸 / 大额成交** | 知道聪明钱方向 | [Unusual Predictions API](https://api.unusualwhales.com/docs) 或自建链上索引（Polygon） |

**为什么必需**：Polymarket 没有定价模型，盘口就是用户共识。我们的 AI 估计概率必须**对标这个共识**，差值就是我们要交付给用户的 edge。

### 4.2 必备：足球比赛数据（AI 预测的输入）

参考 SportBot AI / NerdyTips / Opta 的栈，世界杯场景下需要：

| 数据 | 用途 | 来源建议 |
|------|------|---------|
| **历史比赛结果 + H2H** | 球队基础强度先验 | StatsBomb 开源 / FBref（爬取） |
| **xG / 控球 / 射门 / 传球** | 球队近期真实战斗力（结果之外） | StatsBomb 开源 / FBref / 付费可上 Opta |
| **FIFA / ELO 排名** | 国家队对比的基线（OddsFlow 在用） | [FIFA 官方](https://www.fifa.com/fifa-world-ranking/) / [eloratings.net](https://www.eloratings.net/) |
| **阵容 / 首发预测** | 比赛即时变量（最关键） | Sofascore（爬取）/ 推特公告 / Footystats |
| **伤病 / 停赛** | 比赛即时变量 | 球队官网 + Twitter / Transfermarkt |
| **赛程 / 比赛元数据**（场地/时间/裁判/天气） | 二阶变量；天气和裁判被 SportBot AI 证明有效 | 体育数据 API + OpenWeather |

**为什么必需**：纯靠 LLM 参数化知识做世界杯预测会过时（Willmin Bot 的极简方案在 Kalshi 政治市场行得通，但在体育上靠不住——球员状态变化太快）。

### 4.3 必备：新闻 / 信息流（即时变量）

参考 Polymarket Agents 的 NewsAPI、PolyGnosis 的 GDELT、PolyPulse 的 Perplexity：

| 数据 | 用途 | 来源建议 |
|------|------|---------|
| **结构化新闻** | RAG 喂给 Agent，提供赛前最新信息 | [NewsAPI](https://newsapi.org/) 起步；可叠加 [GDELT](https://www.gdeltproject.org/) |
| **球队/球员官方公告** | 阵容、伤病、内部消息源 | Twitter/X API（球队官号）+ 球队官网 RSS |
| **赛前赔率变动** | sharp action 的领先指标 | Bet365 / Pinnacle / 多盘口聚合（Oddspedia 等） |
| **可选：社交语调 / GDELT tone** | 公众情绪——PolyGnosis 2.0 的"Perspective Mismatch"特征 | GDELT GKG |

**为什么必需**：用户用 AI 而不是直接看 Polymarket，是因为想要**"为什么"**。"为什么"的内容来源就是新闻和信息流。

### 4.4 可选：跨盘口赔率（差异化加成）

| 数据 | 用途 |
|------|------|
| Bet365 / Pinnacle / DraftKings / FanDuel 实时盘口 | 与 Polymarket 价格对比，发现 Polymarket 是否定价错误 |

**为什么可选**：竞品 OddsFlow.ai 的核心功能就是这个；但它只对比传统 sportsbook 之间，**没人做"传统赔率 vs Polymarket 价格"对比**——这是我们的独门 edge。

### 4.5 专题：博彩赔率 vs Polymarket 订单簿——为什么两种都要

两者本质都是"群体对结果的判断"，但机制完全不同，所以信号不同。

**机制差异**

| 维度 | 博彩公司赔率（Bet365 等） | Polymarket 订单簿 |
|------|------------------------|------------------|
| **定价者** | 博彩公司的交易员 / 算法 | 用户自己挂单 |
| **盈利模式** | 抽水（vig，通常 5-10%） | 平台抽手续费（极低） |
| **本质** | "庄家愿意以这个价收你的注" | "买卖双方撮合成交价" |
| **是否反映真实概率** | 不直接——已扣抽水 + 庄家会主动调线管理敞口 | **直接**——价格 = 隐含概率 |
| **谁在下注** | 大众球迷为主，sharp 被限红 | Crypto 用户、量化、对冲基金、政治分析师 |
| **响应速度** | 内部交易员看到消息 → 调线（有延迟） | 任何人看到消息 → 直接挂单（秒级） |

**举例**

> 巴西 vs 摩洛哥，Bet365 巴西赢赔率 1.50（隐含概率 66.7%，扣掉抽水后真实约 64%）
> Polymarket 巴西赢挂单价格 0.62（直接 = 62% 概率）
> → 差距 2 个百分点，Polymarket 比博彩公司"更看好摩洛哥一点"

**为什么两种数据都要**

1. **交叉验证**：两者一致 → 真实共识，置信度高；两者分歧 → 有 edge 或风险
2. **不同人群的判断**：传统盘口反映大众球迷 + 庄家调整；Polymarket 反映 Crypto/量化/分析师群体——后者对足球（尤其非美足球）可能反应慢，是机会
3. **信息传导速度差**：突发消息（赛前阵容变动）Polymarket 反应更快，博彩公司临时关盘或调线慢——中间有套利窗口
4. **AI 各自的用法**：
   - Polymarket 价格 = 我们 AI 要**对标的市场共识基准**
   - 博彩赔率 = **校准 / 参考**，看主流市场怎么想

**判断逻辑示例**

```
AI 估计概率 65%
↓
对比 Polymarket：62% → AI 比市场高 3 个点 → 有 edge，看涨
对比 Bet365：64%（扣抽水后）→ 与 AI 接近，传统市场也认同
↓
高置信度信号：Polymarket 低估了，建议买入
```

反过来：

```
AI 估计概率 65%
对比 Polymarket：62%
对比 Bet365：55%（扣抽水后）→ 两市场分歧很大
↓
先别动：要么 AI 错，要么有我们不知道的信息
```

**结论**

- **Polymarket 订单簿 = 主战场**（AI 直接对标它）
- **博彩公司赔率 = 第二意见**（交叉验证、套利信号）
- 两者结合还能反映 **"Crypto 群体" vs "大众球迷群体"**的判断差异——本身就是有信息量的特征

### 4.5 数据栈分层总结

```
┌─────────────────────────────────────────────┐
│  解释层（用户看到的）                          │
│  - Multi-Agent 共识 + 分歧理由                │
│  - 置信度 % vs Polymarket 市场价             │
└─────────────────────────────────────────────┘
            ↑ AI Agent 调用
┌─────────────────────────────────────────────┐
│  即时变量层（决定预测准不准）                  │
│  - 阵容 / 伤病 / 新闻 / sharp action         │
│  数据源：Twitter/X、Sofascore、NewsAPI       │
└─────────────────────────────────────────────┘
            ↑
┌─────────────────────────────────────────────┐
│  基础数据层（球队强度先验）                    │
│  - xG / 历史 / H2H / FIFA / ELO              │
│  数据源：StatsBomb / FBref / FIFA            │
└─────────────────────────────────────────────┘
            ↑
┌─────────────────────────────────────────────┐
│  市场锚层（对标谁）                            │
│  - Polymarket Gamma + CLOB                   │
│  - 巨鲸/大额成交                              │
│  - 跨盘口赔率（可选）                         │
└─────────────────────────────────────────────┘
```

四层都需要才能形成完整闭环：**市场锚 → 基础数据 → 即时变量 → 解释层**。少了任何一层，要么没法对标（没市场锚就成了通用预测站）、要么不准（没即时变量），要么用户不买账（没解释）。

### 4.6 优先级建议

如果 MVP 阶段要砍：

| 优先级 | 数据 |
|--------|------|
| **P0 必做** | Polymarket CLOB + Gamma、StatsBomb/FBref 历史、FIFA/ELO、阵容/伤病新闻、NewsAPI |
| **P1 重要** | Twitter/X 球队官号、Bet365 实时赔率（对比 Polymarket 用）|
| **P2 加分** | GDELT、巨鲸数据、天气、裁判数据 |
